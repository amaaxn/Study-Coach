## Step-by-Step Instructions

### 1. Navigate to Networking Settings
- In Railway dashboard, go to your service "Learnium"
- Click "Settings" tab (top navigation)
- Click "Networking" in the right sidebar

### 2. Generate Public Domain
- Under "Public Networking" section
- Click the purple **"Generate Domain"** button
- Railway will create a public URL like: `https://learnium-production-xxxx.up.railway.app`

### 3. Copy the URL
- The generated URL will appear in the Public Networking section
- Copy the full URL (including `https://`)

### 4. Use in Frontend

**For Vercel:**
1. Go to Vercel Dashboard ‚Üí Your Project
2. Settings ‚Üí Environment Variables
3. Add:
   - **Key:** `VITE_API_URL`
   - **Value:** `https://learnium-production-xxxx.up.railway.app/api` ‚ö†Ô∏è **Add `/api` at the end**
   - Select all environments
4. Save and redeploy

### 5. Test Your Backend URL

Open in browser:
```
https://your-generated-url.up.railway.app/api/health
```

Should return:
```json
{"status":"ok","environment":"production"}
```

---

## ‚ö†Ô∏è Important Notes

- **The `learnium.railway.internal` URL is PRIVATE** - only works within Railway network
- **You need the PUBLIC URL** - generated by clicking "Generate Domain"
- **Always add `/api`** when using in frontend (`VITE_API_URL`)
- The public URL starts with `https://` and ends with `.up.railway.app`

---

## Alternative: Custom Domain

If you want your own domain instead:
1. Click "Custom Domain" button (next to Generate Domain)
2. Enter your domain (e.g., `api.yourdomain.com`)
3. Configure DNS as instructed
4. Use that domain in `VITE_API_URL`
